# 家庭监控智能助手 - 项目需求文档

## 基于 ESP32-S3-EYE 的人脸识别家庭监控系统

---

## 一、项目概述

基于 **ESP32-S3-EYE** 开发板打造的人脸识别家庭监控智能助手，能够实时捕捉人脸并与后台家庭成员数据库进行比对，识别结果通过日志系统提醒，未来可扩展语音提醒功能。

### 核心功能
- 👤 **人脸识别**：实时检测并识别人脸
- 🏠 **家庭成员比对**：与后台人员数据匹配
- 📋 **日志提醒**：记录所有识别事件并推送通知
- 🔊 **语音扩展**：预留扬声器接口支持声音提醒（V1.1）

## 三、功能模块详细设计

### 3.1 ESP32-S3-EYE 固件端

| 功能 | 描述 |
|------|------|
| 人脸检测 | 使用 ESP-DL 库进行实时人脸检测 |
| 人脸特征提取 | 提取人脸特征向量（128/512维） |
| 图像采集 | 定时/触发式拍照（QVGA/VGA分辨率） |
| WiFi连接 | 连接家庭WiFi网络 |
| 数据上传 | 通过HTTP/MQTT将图片/特征上传服务器 |
| 状态反馈 | LED指示识别状态（已识别/未知/错误） |
| 心跳机制 | 定期向服务器发送心跳包 |
| OTA升级 | 支持远程固件升级 |

### 3.2 后端服务

| 功能 | 描述 |
|------|------|
| API接口 | RESTful API接收设备数据 |
| 人脸比对 | 计算特征向量相似度，判断是否家庭成员 |
| 用户管理 | 添加/编辑/删除家庭成员人脸数据 |
| 日志记录 | 记录所有识别事件和系统日志 |
| 通知提醒 | 邮件/短信/Webhook通知异常识别 |
| 数据统计 | 识别频率、时间段分析等 |

### 3.3 前端管理面板

| 功能 | 描述 |
|------|------|
| 实时监控 | 查看实时抓拍图片（WebRTC/WSS） |
| 人员管理 | 管理家庭成员信息、上传人脸照片 |
| 日志查询 | 按时间/人员/类型筛选识别日志 |
| 系统设置 | WiFi配置、通知设置、灵敏度调节 |
| 数据统计 | 可视化图表展示识别数据 |

## 五、数据模型设计

### 5.1 家庭成员表 (family_members)

```sql
CREATE TABLE family_members (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(100) NOT NULL,
    avatar_image_path VARCHAR(255),
    face_embedding BLOB NOT NULL,  -- 人脸特征向量
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    is_active BOOLEAN DEFAULT TRUE,
    notes TEXT
);
```

### 5.2 识别日志表 (recognition_logs)

```sql
CREATE TABLE recognition_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    member_id INTEGER,              -- 如果识别为家庭成员
    confidence REAL,                 -- 识别置信度
    matched BOOLEAN,                 -- 是否匹配家庭成员
    image_path VARCHAR(255),         -- 抓拍图片存储路径
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    device_id VARCHAR(50),
    note TEXT,                       -- 备注（如：陌生人报警）
    FOREIGN KEY (member_id) REFERENCES family_members(id)
);
```

### 5.3 系统配置表 (system_config)

```sql
CREATE TABLE system_config (
    key VARCHAR(100) PRIMARY KEY,
    value TEXT,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 七、工作流程

```
┌──────────────────────────────────────────────────────────────┐
│                      人脸识别流程                             │
└──────────────────────────────────────────────────────────────┘

  ① 摄像头捕获图像
        │
        ▼
  ② 人脸检测（是否有脸）
        │
        ├─ 无人脸 → 返回待机
        │
        ▼
  ③ 人脸质量评估（清晰度、角度）
        │
        ├─ 质量差 → 丢弃/重试
        │
        ▼
  ④ 特征提取 → 生成特征向量
        │
        ▼
  ⑤ 上传服务器（图片+设备ID+时间戳）
        │
        ▼
  ⑥ 服务器接收并处理
        │
        ▼
  ⑦ 与数据库中人脸特征逐一比对
        │
        ├─ 最高相似度 > 阈值(0.6) → 匹配成功
        │      │
        │      ▼
        │   识别为已知成员
        │      │
        │      ▼
        │   记录日志 + 发送通知（可选）
        │
        └─ 最高相似度 ≤ 阈值 → 匹配失败
               │
               ▼
            记录为陌生人
               │
               ▼
            发送报警通知
        │
        ▼
  ⑧ 返回识别结果给设备
        │
        ▼
  ⑨ 设备LED指示（绿色=家人/红色=陌生人）
```

## 十、关键参数建议

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 识别阈值 | 0.55 - 0.65 | 平衡误识率/拒识率 |
| 检测间隔 | 5-30秒 | 可调，避免过度消耗资源 |
| 图片分辨率 | 160x120 (QVGA) | 人脸区域足够识别 |
| 上传图片 | 320x240 (QVGA) | 保留完整场景 |
| 特征维度 | 128维 | 平衡精度与传输 |
| LED指示 | 绿色=家人/红色=陌生人/蓝色=检测中 | |

## 十四、注意事项

1. **隐私保护**：人脸数据属于敏感信息，需加密存储
2. **网络稳定**：确保家庭WiFi覆盖良好
3. **电源管理**：建议持续供电，使用UPS备用电源
4. **光线条件**：安装位置需有合适的光照条件
5. **摄像头角度**：覆盖入口/门口最佳位置
6. **识别精度**：初期需收集数据调优阈值参数
